Give an LLM a Sui wallet.

KAIROS hands a language model a wallet, a name, and a set of actions. It signs PTBs, swaps on DeepBook, keeps its memory on Walrus, and reads any object on chain. You ask in plain language. It does the work and shows you the digest.

Build an agent ↗ See how it works

Other chains have Eliza, AgentKit, a dozen frameworks. Sui had none.

So we built the one that speaks Move natively. Four things make it different.

PTB_NATIVE

Multi-step, or nothing

Transfer, swap, stake, read objects. A swap-then-transfer is one transaction. It lands whole or it doesn't land. No half-finished state to clean up, which is the part EVM agents never quite solved.

WALRUS_MEMORY

Memory it actually owns

The agent's memory sits on Walrus, not in some database you rent. Kill the process, boot it back up tomorrow, and it still knows what it did. The user owns the blobs. Nobody can quietly delete them.

20_LINES

Actions are tiny

Write a TypeScript object. The model reads the description and decides when to call it. No router. No parser. No state machine you have to babysit.

MULTI_AGENT

Agents that coordinate

One agent writes to Walrus. Another reads it back as context. They talk through the chain, never through each other's APIs, so there's no shared secret to leak.

CLAUDE

Real tool use

Every action becomes a Claude tool. The model picks what to call, you execute it, the result feeds back. That loop is the whole framework.

Running in three commands

Scaffold it, drop in your keys, talk to it over HTTP.

# scaffold a new agent
npx create-kairos-agent Atlas

# add your keys, then boot it
cd agents/atlas && cp .env.example .env
pnpm install && pnpm start

# ask it something
curl -X POST localhost:3000/message \
  -H "Content-Type: application/json" \
  -d '{"text": "what is my SUI balance?", "userId": "alice"}'
